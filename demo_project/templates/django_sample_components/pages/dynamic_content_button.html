Modal response

The server returns a Bootstrap modal + a script that opens it. The modal is not in the page HTML — it is injected on demand:

How It Works
  1. Click the button — HTMX sends a GET to content_url
  2. The server response is appended to <body> via hx-swap="beforeend"
  3. If the response contains a <script>, the browser runs it immediately
  4. If the response contains a modal, the script in the response opens it
Server-driven UX

The button itself is generic — it knows only the URL to call. The server decides what happens: run a confirmation dialog, open a detail modal, show a delete-blocked warning, inject a dynamic form, etc. This is the same pattern used in Kapybar's organisation delete flow.
